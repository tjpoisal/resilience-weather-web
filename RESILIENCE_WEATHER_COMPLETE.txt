DATABASE SCHEMA (8 TABLES)
═════════════════════════════════════════════════════════════════════════════

1. users
   └─ id (UUID)
   └─ email, name
   └─ stripe_customer_id, stripe_subscription_id
   └─ plan (free/pro/enterprise)
   └─ operational_profile
   └─ preferences (JSONB)
   └─ created_at, updated_at

2. observations
   └─ user_id (FK to users)
   └─ latitude, longitude
   └─ temperature, humidity, wind_speed
   └─ conditions, notes
   └─ verified (boolean)
   └─ created_at

3. subscriptions
   └─ user_id (FK to users)
   └─ latitude, longitude
   └─ alert_types (text array)
   └─ push_enabled, email_enabled, sms_enabled
   └─ notification_frequency
   └─ created_at, deleted_at

4. alerts_cache
   └─ noaa_id (from NOAA API)
   └─ event, headline, description
   └─ severity, effective, expires
   └─ properties (JSONB)

5. stripe_events
   └─ event_id (unique)
   └─ event_type (payment_intent.succeeded, etc.)
   └─ user_id (FK)
   └─ stripe_customer_id, stripe_subscription_id
   └─ data (JSONB)
   └─ processed (boolean)

6. audit_log
   └─ user_id (FK)
   └─ action (user_created, observation_submitted, etc.)
   └─ resource_type, resource_id
   └─ details (JSONB)
   └─ ip_address, user_agent
   └─ created_at

7. operational_profiles
   └─ name (15 profiles: General Public, Marine, Agricultural, etc.)
   └─ description
   └─ alert_categories (text array)
   └─ mesh_networking, offline_first

8. audit_log (triggers)
   └─ Automatic logging on user updates
   └─ Automatic logging on observation submissions
   └─ Federal compliance audit trail

API ENDPOINTS (LIVE AFTER EB READY)
═════════════════════════════════════════════════════════════════════════════

HEALTH & DIAGNOSTICS:
  GET /api/health
  └─ Returns server status, timestamp, uptime

WEATHER DATA:
  GET /api/weather/alerts?latitude=38.9&longitude=-77.0
  └─ Fetches NOAA active alerts for location

  GET /api/weather/forecast?latitude=38.9&longitude=-77.0
  └─ Returns 7-day forecast with confidence scores

OBSERVATIONS:
  POST /api/observations
  └─ Submit weather observation from user
  
  GET /api/observations/:userId
  └─ Retrieve user's observations

SUBSCRIPTIONS:
  POST /api/notifications/subscribe
  └─ Subscribe user to location-based alerts
  
  DELETE /api/notifications/subscribe/:subscriptionId
  └─ Unsubscribe from alerts

STRIPE PAYMENTS:
  POST /api/webhook/stripe
  └─ Webhook handler for Stripe events
  
  POST /api/checkout
  └─ Create Stripe checkout session

USER PROFILES:
  GET /api/user/profile/:userId
  └─ Get user profile with plan status
  
  PUT /api/user/profile/:userId
  └─ Update user preferences
